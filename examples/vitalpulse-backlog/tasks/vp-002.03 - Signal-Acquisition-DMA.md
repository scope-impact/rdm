## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
As a **Firmware engineer**, I want to **implement DMA-based signal acquisition for SpO2** so that I can **ensure deterministic sample timing without CPU intervention**.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria

<!-- AC:BEGIN -->
- [ ] #1 DMA transfers ADC samples directly to circular buffer
- [ ] #2 Sample rate maintained at 500Hz +/- 0.1%
- [ ] #3 Double-buffering prevents data loss during processing
- [ ] #4 DMA complete interrupt triggers data-ready event
- [ ] #5 CPU utilization <5% for sample acquisition
- [ ] #6 Jitter <100us between consecutive samples
<!-- AC:END -->

## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
- **DMA:** STM32H7 MDMA for high-bandwidth transfers
- **Buffer:** Dual 256-sample buffers (512ms each)
- **Why:** Deterministic timing critical for SpO2 ratio calculation
- **Dependencies:** Requires FreeRTOS queue for buffer handoff (vp-001.03)
<!-- SECTION:NOTES:END -->
